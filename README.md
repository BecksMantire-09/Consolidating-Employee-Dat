I was hired as the first data practitioner in a fast-growing small business where HR data was scattered across multiple teams and stored in different formats including CSV, Excel, and JSON files. The lack of structure made it difficult to analyze employee information and support decision-making.

My first mission was to bring structure and consistency to the HR data. I began by loading and inspecting all datasets using Python (Pandas), identifying inconsistencies in formats, missing headers, and mismatched structures across files.

I then standardized the datasets by:

Reading and correctly importing CSV, Excel, and JSON files
Cleaning column names and handling missing values
Renaming inconsistent headers for clarity and uniformity
Converting unstructured JSON data into a tabular format
Merging employee, emergency contact, and office datasets using appropriate keys
Handling mismatches using correct join strategies (left joins where necessary to preserve employee records)
Setting employee_id as the primary index for structured access

Finally, I produced a clean, unified HR dataset that combines employee details, job roles, emergency contacts, and office information into a single structured DataFrame suitable for analysis and reporting.

This work established a standardized HR data foundation that enables reliable reporting, future automation, and scalable analytics for the company’s continued growth.
